class whylogs.core.metrics.metrics.CustomMetricBase[source]#

Bases: Metric, abc.ABC

You can use this as a base class for custom metrics that dont use the supplied or custom MetricComponents. Subclasses must be decorated with @dataclass. All fields not prefixed with an underscore will be included in the summary and will be [de]serialized. Such subclasses will need to implement the namespace, merge, and zero methods. They should be registered by calling register_metric()
