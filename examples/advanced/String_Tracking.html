String Tracking - Unicode Range and String Length#

By default, columns of type str will have the following metrics, when logged with whylogs: - Counts - Types - Frequent Items/Frequent Strings - Cardinality

We will do that by counting, for each string record, the number of characters that fall in a given unicode range, and then generating distribution metrics, such as mean, stddev and quantile values based on these counts. In addition to specific unicode ranges, weâ€™ll do the same approach, but for the overall string length.

In this example, weâ€™re interested in tracking two specific ranges of characters: - ASCII Digits (unicode range 48-57) - Latin alphabet (unicode range 97-122)

Configuring the Metrics in the DatasetSchema#

whylogs uses Resolvers in order to define the set of metrics tracked for a column name or data type. In this case, weâ€™ll create a custom Resolver to apply the UnicodeRangeMetric to all of the columns.

If youâ€™re interested in seeing how you can add or remove different metrics according to the column type or column name, please refer to this example on Schema Configuration

[3]:

from whylogs.core.schema import ColumnSchema, DatasetSchema
from whylogs.core.metrics.unicode_range import UnicodeRangeMetric
from whylogs.core.resolvers import Resolver
from whylogs.core.datatypes import DataType
from typing import Dict
from whylogs.core.metrics import Metric, MetricConfig

class UnicodeResolver(Resolver):
    def resolve(self, name: str, why_type: DataType, column_schema: ColumnSchema) -> Dict[str, Metric]:
        return {UnicodeRangeMetric.get_namespace(): UnicodeRangeMetric.zero(column_schema.cfg)}

Resolvers are passed to whylogs through a DatasetSchema, so weâ€™ll have to create a custom Schema as well.

Weâ€™ll just have to: - Pass the UnicodeResolver created previously - Since weâ€™re interested in changing the default character ranges, weâ€™ll also pass a Metric Configuration with the desired ranges

[4]:

schema = DatasetSchema(resolvers=UnicodeResolver(), default_configs=config)

If a default MetricConfig is not passed, it would use the default unicode ranges, which would track the default ranges such as: emoticons, control characters and extended latin.

We can now log the dataframe and pass our schema when calling log:

[5]:

import whylogs as why

prof_results = why.log(df, schema=DatasetSchema(resolvers=UnicodeResolver(), default_configs=MetricConfig(unicode_ranges={"digits": (48, 57), "alpha": (97, 122)})))
prof = prof_results.profile()

You can see thereâ€™s a lot of different metrics for each of the original dataframeâ€™s columns. In the unicode_range metric, weâ€™ll have additional sub metrics. In this case, we have metrics for:

  • digits: distribution metrics for characters inside the unicodeâ€™s digit range

  • alpha: distribution metrics for characters inside the unicodeâ€™s lowercase letters range

  • UNKNOWN: distribution metrics for character that fall anywhere outside the predefined range (digits and alpha)

  • string_length: distribution metrics for overall string length

For each of these submetrics, we have metric components such as:

  • mean: the calculated mean for the column

  • stddev: the calculated standard deviation for the column

  • n: the total number of record for the column

  • max, min: maximum and minimum values for the column

  • q_xx: the xx-th quantile value of the dataâ€™s distribution

  • median: the median for the column
